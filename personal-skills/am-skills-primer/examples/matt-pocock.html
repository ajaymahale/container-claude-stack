Two rules that matter:
1. Keep discuss → spec → plan in one unbroken context window. Don't /compact or clear before /to-issues.
2. /triage is only for issues you didn't write. /to-issues output is already agent-ready — don't re-triage it.

/handoff forks (new session, old preserved). /compact continues (same session). Handoff when near the smart-zone ceiling (~120k tokens) mid-flow.

Engineering — the flow skills

/ask-matt router
You don't remember which skill fits. Routes you over the user-invoked skills.
/setup-matt-pocock-skills setup
First-time config of tracker / label vocabulary / domain doc layout. Run once.
/grill-with-docs discuss
Codebase + want the interview to also produce ADRs and a glossary as you go.
/grill-me discuss
Stress-test a plan with no codebase / nothing to persist. Stateless twin of grill-with-docs.
/prototype discuss → detour
A design question is faster answered by running code — a terminal app for state/business-logic, or several toggleable UI variations from one route.
/to-prd spec
Conversation is settled → turn it into a PRD and publish to the tracker.
/to-issues plan
A PRD/plan needs slicing into independently-grabbable issues via tracer-bullet vertical slices.
/triage on-ramp
Inbound bugs / external PRs you didn't author are piling up. Moves them through triage roles into agent-ready briefs.
/implement execute
Build a specific issue or PRD — fresh session per issue.
tdd execute / test
Build feature/fix test-first. Fires on "red-green-refactor" or a request for integration tests.
diagnosing-bugs test / debug
Something's broken/throwing/slow and the cause isn't obvious. Say "diagnose" / "debug this".
codebase-design execute
Designing or improving a module's interface — deep modules, where a seam goes, making code testable/AI-navigable.
domain-modeling execute
Pin down domain terminology / ubiquitous language, or record an architectural decision.
/improve-codebase-architecture upkeep
Periodic (every few days). Scans for deepening opportunities → visual HTML report → grills the one you pick.
resolving-merge-conflicts push
An in-progress git merge/rebase has conflicts to resolve.
