One-line verdict: GSD = how the project runs (durable, orchestrated, gated). Addy = how a single concern is done well (stateless, opinionated, portable). Keep GSD as spine; reach for Addy per-concern.

All 24 skills — by phase

meta   using-agent-skills
Session start. Routes incoming work → right skill. Defines the pack's shared operating rules. The router.
define   interview-me
One-question-at-a-time interview → ~95% confidence on real intent. Stops you silently filling in ambiguous asks. Fires on "interview me" / "grill me" / "are we sure?".
define   idea-refine
Divergent then convergent thinking. Vague idea → sharp proposal. "ideate" / "refine this idea" / "stress-test my plan".
define   spec-driven-development
Write a PRD (objectives, commands, structure, style, testing, boundaries) before any code.
plan   planning-and-task-breakdown
Decompose a spec → small verifiable tasks with acceptance criteria + dependency ordering.
build   incremental-implementation
Thin vertical slices: implement → test → verify → commit. Feature flags, safe defaults, rollback-friendly. Any change >1 file.
build   test-driven-development
Red-Green-Refactor. Test pyramid 80/15/5, DAMP over DRY, Beyoncé Rule.
build   context-engineering graft
Feed the agent the right info at the right time — rules files, context packing, MCP. Fires on new session / degrading output / switching tasks.
build   source-driven-development graft
Ground every framework decision in official docs — verify, cite, flag unverified. Pairs with generator/Protobuf/Mongo decisions.
build   doubt-driven-development graft
The one to steal. Adversarial fresh-context review of every non-trivial decision: CLAIM → EXTRACT → DOUBT → RECONCILE → STOP. Optional cross-model escalation. Fits byte-identical binding gates.
build   frontend-ui-engineering skip
Component arch, design systems, state, responsive, WCAG 2.1 AA. Web-only — no UI in this stack.
build   api-and-interface-design graft
Contract-first, Hyrum's Law, One-Version Rule, error semantics, boundary validation. For legacy reverse-map / IArticleRepository seams.
verify   browser-testing-with-devtools skip
Chrome DevTools MCP — DOM, console, network, profiling. Web-only, needs chrome-devtools MCP.
verify   debugging-and-error-recovery
5-step triage: reproduce → localize → reduce → fix → guard. Stop-the-line rule, safe fallbacks.
review   code-review-and-quality
Five-axis review, change sizing (~100 lines), severity labels (Nit/Optional/FYI), splitting strategies. Before any merge.
review   code-simplification graft
Chesterton's Fence, Rule of 500. Reduce complexity, preserve exact behavior. Good on the LegacyMapperGenerator.
review   security-and-hardening
OWASP Top 10, auth patterns, secrets mgmt, dep auditing, three-tier boundary system.
review   performance-optimization low
Measure-first — Core Web Vitals, profiling, bundle analysis. Web-leaning; backend perf needs other tooling.
ship   git-workflow-and-versioning
Trunk-based, atomic commits, change sizing (~100 lines), commit-as-save-point. Every change.
ship   ci-cd-and-automation
Shift Left, Faster-is-Safer, feature flags, quality-gate pipelines, failure feedback loops.
ship   deprecation-and-migration graft
Code-as-liability mindset, compulsory vs advisory deprecation, migration patterns, zombie-code removal. Your PIES/rename work is exactly this.
ship   documentation-and-adrs
ADRs, API docs, inline-doc standards — document the why.
ship   observability-and-instrumentation graft
Structured logging, RED metrics, OpenTelemetry tracing, symptom-based alerting. Instrument as you build. For API + publishers.
ship   shipping-and-launch
Pre-launch checklists, feature-flag lifecycle, staged rollouts, rollback, monitoring setup.

For your stack (.NET backend, already on GSD) — graft, don't replace

High-value to graft in:
• doubt-driven-development — fits your byte-identical generator gates perfectly
• source-driven-development — cite official docs for generator/Protobuf/Mongo decisions
• api-and-interface-design — contract-first for the legacy reverse-map seams
• deprecation-and-migration — PIES/rename work is deprecation territory
• observability-and-instrumentation — RED/OTel for API + publishers
• code-simplification — Chesterton's Fence on LegacyMapperGenerator
• The 4 personas — run agent-skills:code-reviewer / security-auditor standalone on PRs
Skip (web-only, no browser app here): browser-testing-with-devtools, frontend-ui-engineering, performance-optimization, webperf.
